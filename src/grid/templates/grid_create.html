Atualment existem duas opções para inserir uma versão da grade no sistema:

Enviar a grade no formato JSON Gerador de grade (em fase de teste)



O conteúdo do json deve seguir um formato com as seguintes especificações:

Descrição das chaves e valores


  • "version": Nome da versão da grade (até 40 caracteres). Recomenda-se inserir o ano relativo da grade no curso.
  • "grid": Matriz com os códigos de cada disciplina para montar a grade. Cada linha representa um novo período da grade.
  • "repeated_codes": Lista de códigos que se repetem na grade.
    Caso este campo esteja vazio, o servidor tentará automaticamente preencher com os valores corretos, mas recomenda-se que o usuário já informe os códigos repetidos.
  • "fake_codes": Lista de códigos que são falsos. Alguns códigos da grade pode representar um conjunto de disciplinas, por exemplo uma disciplina optativa.
    Dessa forma, é necessário indicar que essas disciplinas não irão ser presentes nas planilhas submetidas no sistema e evitar links inválidos nas páginas de visualização dos relatórios gerados pelo sistema.
  • "code_to_name": Mapeamento do código de cada disciplina para o seu nome completo.
  • "equiv_codes": Mapeamento do código de cada disciplina para uma lista que contém todas os códigos de disciplinas que são suas equivalentes na grade. Isso pode ser necessário para visualizar as regras de migração de grade aceitas em colegiado.
    Além disso, caso o usuário adicione uma disciplina com "código falso" na grade, é provável que ele queira adicionar as equivalências aqui (por exemplo todas as disciplinas que se relacionam com o código relativo às disciplinas optativas).
    Esses dados serão utilizados nas análises para identificar algumas informações, por exemplo o período de cada aluno em relação à grade.
  • "prerequisites": Mapeamento do código de cada disciplina para uma lista de códigos de disciplinas que representam seus pré-requisitos.
    Essa informação será utilizada no processamento dos dados para identificar por exemplo quais disciplinas o aluno está apto a se matricular no próximo semestre.
    No caso de situações como a "Barreira", onde todas as disciplinas após o terceiro período são requisitos das disciplinas anteriores ao quarto, é necessário informar todas as dependências como mostrado no exemplo ao lado.
  • "phases": Mapeamento de nome de fases (pode ser qualquer nome) para uma lista de códigos pertecentes à grade descrita no outro campo.
    Esses valores são utilizados para processar os dados e por exemplo gerar listagens dos alunos que já cursaram essas fases da grade. Isso pode ser útil para identificar por exemplo os alunos que já cursaram a "Barreira" ou os alunos que já cursaram todas disciplinas de matemática.
